Vue d'ensemble du projet

ÉduRéussite QC est une plateforme éducative intelligente destinée aux élèves québécois (primaire et secondaire). Elle génère des exercices personnalisés par l'IA (Claude d'Anthropic), fournit un feedback pédagogique détaillé, crée automatiquement des cours de remédiation après les erreurs, et offre un tableau de bord aux parents pour suivre la progression de leurs enfants.

🌐
Application en productionURL : edureussite.edevtic.com · Hébergé sur Vercel · Base de données Supabase (PostgreSQL)

Installation locale

Mettre en place le projet en local, étape par étape.

Cloner et installer

  1. Récupérer le code source
    terminal
    git clone [URL_DU_REPO] edureussite-qc
cd edureussite-qc
  2. Installer les dépendances
    terminal
    npm install
  3. Créer le fichier de variables d'environnement
    Créer un fichier .env à la racine du projet avec ce contenu (voir section 11 pour les valeurs) :
    .env
    # Base de données Supabase
DATABASE_URL="postgresql://..."

# NextAuth
NEXTAUTH_SECRET="une-chaine-aleatoire-longue"
NEXTAUTH_URL="http://localhost:3000"

# Claude AI (Anthropic)
ANTHROPIC_API_KEY="sk-ant-api03-..."

# Resend (emails)
RESEND_API_KEY="re_..."
EMAIL_FROM="noreply@votre-domaine.com"
  4. Générer le client Prisma
    terminal
    npx prisma generate
    ⚠️
    ImportantLe client Prisma est généré dans src/generated/prisma/ (pas le chemin par défaut). C'est configuré dans prisma/schema.prisma avec output = "../src/generated/prisma".
  5. Synchroniser la base de données
    Si c'est une première installation (nouvelle base de données vide) :
    terminal
    npx prisma db push
  6. Lancer le serveur de développement
    terminal
    npm run dev
    L'application est accessible sur http://localhost:3000

Base de données (Supabase + Prisma)

Créer le projet Supabase

  1. Créer un compte sur supabase.com et créer un nouveau projet.
  2. Dans le dashboard Supabase, aller dans Settings → Database → Connection string.
  3. Copier l'URL de connexion Transaction mode (port 6543) — c'est celle-ci qui doit être utilisée en production pour éviter l'erreur MaxClientsInSessionMode avec PgBouncer.
    ⚠️
    Attention aux ports
    Port 5432 = connexion directe (pour le développement local)
    Port 6543 = PgBouncer transaction mode (pour Vercel en production)
    Toujours utiliser le port 6543 en production. Ajouter aussi ?sslmode=disable si Supabase l'exige.
    .env
    DATABASE_URL="postgresql://postgres.[PROJECT-REF]:[PASSWORD]@aws-1-ca-central-1.pooler.supabase.com:6543/postgres?sslmode=disable"

Authentification

L'authentification utilise NextAuth.js avec un provider Credentials personnalisé. Le fichier de configuration est src/lib/auth.ts.

Flux de connexion — Adultes (Parent, Enseignant, Admin)

  1. L'utilisateur entre son email + mot de passe sur /login
  2. Le backend vérifie les credentials via trpc.auth.initiateLogin
    → Si email non vérifié : envoie un OTP de vérification et redirige vers /verify-email
    → Si email vérifié : envoie un OTP de connexion et affiche les champs OTP
  3. L'utilisateur entre le code à 6 chiffres reçu par email
  4. NextAuth crée la session et redirige vers le bon dashboard selon le rôle

Flux de connexion — Élève

  1. L'élève entre son code d'accès (ex: Charbel-1234) + mot de passe
  2. Pas d'OTP — connexion directe. Les élèves n'ont pas d'adresse email réelle.

OTP — Comment ça marche

Les codes OTP sont stockés dans la table verification_tokens avec une expiration. La génération et la vérification se trouvent dans src/server/routers/auth.ts :

typescript
// Créer un OTP (6 chiffres aléatoires, expire dans 10 minutes)
async function createOtp(email: string) { ... }

// Vérifier un OTP
async function verifyOtp(email: string, otp: string): Promise<boolean> { ... }

Générer un NEXTAUTH_SECRET

terminal
openssl rand -base64 32    # Sur Mac/Linux
# Sur Windows : utiliser https://generate-secret.vercel.app/32

Nom de domaine personnalisé

Le domaine edureussite.edevtic.com est un sous-domaine d'edevtic.com, géré chez o2switch. La configuration relie o2switch → Vercel.

Étape 1 — Ajouter le domaine sur Vercel

  1. Aller sur vercel.com → sélectionner le projet → onglet Settings → Domains
  2. Entrer le sous-domaine : edureussite.edevtic.com → cliquer Add
  3. Vercel affiche une erreur car le DNS n'est pas encore configuré. Il indique la valeur CNAME à utiliser : cname.vercel-dns.com

Étape 2 — Configurer le DNS sur o2switch

  1. Se connecter au cPanel o2switch → section Domaines → Zone Editor
  2. Cliquer sur Gérer à côté du domaine edevtic.com
  3. Ajouter un nouvel enregistrement DNS :
    TypeNomValeur
    CNAMEedureussite.edevtic.com. (avec le point final)cname.vercel-dns.com
  4. Sauvegarder l'enregistrement. La propagation DNS peut prendre 5 à 30 minutes (parfois jusqu'à 24h).
  5. Retourner sur Vercel → onglet Domains → le statut passe de ❌ à ✅ une fois propagé.
✅
Comment vérifier que ça marche ?
Ouvrir un terminal et taper : nslookup edureussite.edevtic.com
Si la réponse mentionne vercel, la propagation est effectuée.

Étape 3 — Mettre à jour NEXTAUTH_URL

terminal
vercel env rm NEXTAUTH_URL production --yes
echo "https://edureussite.edevtic.com" | vercel env add NEXTAUTH_URL production
vercel --prod   # redéployer pour prendre en compte

Emails avec Resend

Les emails OTP (codes de connexion et de vérification) sont envoyés via Resend. Sans vérification du domaine, Resend ne peut envoyer qu'à l'adresse de votre propre compte (mode test).

⚠️
Mode test vs Production
En mode test (sans domaine vérifié), tous les emails vont à l'adresse du compte Resend uniquement — les vraies utilisateurs ne reçoivent rien. La vérification du domaine est indispensable.

Étape 1 — Créer le compte et la clé API

  1. Créer un compte sur resend.com
  2. Aller dans API Keys → Create API Key → copier la clé re_xxxx
  3. Ajouter dans Vercel : RESEND_API_KEY=re_xxxx

Étape 2 — Ajouter et vérifier le domaine

  1. Sur Resend → section Domains → Add Domain → entrer edevtic.com
  2. Resend affiche 3 enregistrements DNS à ajouter sur o2switch. Aller dans cPanel → Zone Editor → Gérer edevtic.com et ajouter :
    TypeNomValeur
    TXTresend._domainkey.edevtic.comClé DKIM fournie par Resend (commence par p=MIGf...)
    MXsend.edevtic.comValeur feedback SES fournie par Resend · Priorité : 10
    TXTsend.edevtic.comv=spf1 include:amazonses.com ~all
  3. Attendre 5-15 minutes, puis cliquer Verify DNS Records sur Resend. Le statut passe à Vérifié ✅.

Étape 3 — Mettre à jour EMAIL_FROM

Une fois le domaine vérifié, mettre à jour la variable d'envoi :

terminal
vercel env rm EMAIL_FROM production --yes
echo "noreply@edevtic.com" | vercel env add EMAIL_FROM production
vercel --prod
💡
Adresse d'expéditeur
Utiliser noreply@edevtic.com (domaine racine vérifié), pas noreply@send.edevtic.com. Seul le domaine racine est vérifié sur Resend dans cette configuration.

Fonctionnalités clés

Génération d'exercices par l'IA

Chaque exercice est généré par Claude (claude-sonnet-4-6) en fonction du profil de l'élève (prénom, niveau scolaire, intérêts, style d'apprentissage). Le code se trouve dans src/lib/ai/exercice.ts.

Une validation automatique vérifie que l'exercice contient bien des questions. Si ce n'est pas le cas, la génération est relancée automatiquement avec un prompt correctif (max 2 tentatives).

Cours de remédiation automatiques

Après chaque tranche de 3 exercices terminés, un cours personnalisé est généré automatiquement. Fichier : src/lib/ai/cours.ts.

⚠️
Tâche background sur Vercel
La génération de cours utilise waitUntil() de @vercel/functions. Sans ça, la fonction serverless serait tuée dès l'envoi de la réponse, avant que le cours soit sauvegardé. Ne jamais utiliser un simple .then() sans await pour des tâches longues sur Vercel.

Overlay anniversaire

Si la date de naissance de l'élève correspond à aujourd'hui, un overlay plein écran avec des confettis et une animation Framer Motion s'affiche au chargement du dashboard. Fichier : src/components/eleve/birthday-overlay.tsx.

Gestion des rôles (Admin / Super Admin)

Un SUPER_ADMIN peut promouvoir un PARENT ou ENSEIGNANT en ADMIN, ou rétrograder un ADMIN. Un ADMIN ne peut pas modifier d'autres admins ni le Super Admin. Logique dans src/server/routers/admin.ts et bouton dans src/components/admin/changer-role-btn.tsx.

Show/Hide mot de passe

Le composant src/components/ui/input.tsx détecte automatiquement type="password" et ajoute une icône œil pour afficher/masquer le mot de passe. Aucune modification nécessaire pour les champs qui utilisent ce composant.

Responsive mobile

Toutes les navbars ont un menu hamburger sur mobile (< 768px) qui se ferme au clic extérieur ou au changement de route. Les grilles passent d'une à deux colonnes selon la taille d'écran avec les classes Tailwind sm: et lg:.

Session avec Sofia — Enseignant IA

Chaque cours de remédiation dispose d'un deuxième mode : Session avec Sofia, une enseignante IA vocale interactive. L'interface se compose d'une colonne avatar (photo, contrôles audio, timer) et d'une zone de conversation.

  • TTS (Text-to-Speech) : ElevenLabs eleven_multilingual_v2 via /api/avatar/speak. La clé ELEVENLABS_API_KEY reste strictement côté serveur.
  • IA Chat : Claude claude-sonnet-4-6 via /api/avatar/chat. Prompt système 4 personas (enseignante + psychoneurologue + orthopédagogue + coach).
  • Microphone : Web Speech API (SpeechRecognition / webkitSpeechRecognition). Transcription temps réel dans le champ de saisie (continuous: true, interimResults: true). Aucun audio stocké (conformité Loi 25).
  • Timer de session : Quota quotidien 60 min stocké en localStorage (clé sessionIA_YYYY-MM-DD). Composant : src/components/cours/enseignant-ia-timer.tsx.
  • Résumé parent : En fin de session, generateSummary: true est envoyé à /api/avatar/chat qui crée une notification de type RAPPORT_POSITIF en base via Prisma.
  • Déverrouillage audio iOS : Un son silencieux est joué sur le premier clic micro via unlockAudio() dans src/hooks/useAvatarVoice.ts pour contourner la politique autoplay d'iOS.

Fichiers principaux : src/components/cours/enseignant-ia.tsx, src/hooks/useAvatarVoice.ts, src/lib/avatarVoices.ts, src/app/api/avatar/speak/route.ts, src/app/api/avatar/chat/route.ts.

Déconnexion automatique par inactivité

Le composant src/components/providers/idle-timeout.tsx détecte l'inactivité de l'utilisateur connecté. Après 13 minutes sans interaction (souris, clavier, clic, scroll, touch), un modal s'affiche avec un compte à rebours de 2 minutes. Si l'utilisateur ne réagit pas, signOut() de NextAuth est appelé et l'utilisateur est redirigé vers /login. Tout mouvement réinitialise le timer. Le composant est monté dans src/app/layout.tsx et ne s'active que si une session est présente.

Configuration : IDLE_MS = 15 * 60 * 1000 (15 min total) · WARN_MS = 2 * 60 * 1000 (avertissement à 2 min de la fin). La session JWT a une durée max absolue de 8h (maxAge: 8 * 60 * 60 dans src/lib/auth/config.ts).

Lier un enfant existant (code d'accès)

Le formulaire "Lier un enfant existant" dans le tableau de bord parent demande le code d'accès de l'enfant, et non son courriel. Fichier : src/components/parent/ajouter-enfant-modal.tsx. Les élèves n'ont pas de courriel — leur identifiant est le code d'accès généré à la création du compte.

Journaux de sécurité (Security Logging)

Le système enregistre automatiquement les événements sensibles dans un modèle SecurityLog en base de données. L'utilitaire logSecurityEvent() est fire-and-forget : il ne bloque jamais les opérations principales (enveloppé dans un try/catch silencieux).

Modèle Prisma

prisma
// prisma/schema.prisma
enum SecuritySeverity { INFO  WARNING  CRITICAL }

model SecurityLog {
  id         String           @id @default(cuid())
  createdAt  DateTime         @default(now())
  action     String
  severity   SecuritySeverity
  userId     String?          // FK nullable (null si non authentifié)
  userEmail  String?
  userRole   String?
  cibleId    String?
  cibleEmail String?
  ip         String?
  details    Json
}

Tagging des documents IA par matière et niveau

Les DocumentPedagogique peuvent être associés à des matières (Matiere[]) et des niveaux scolaires (NiveauScolaire[]). Ces tags sont utilisés par getContexteDocuments() pour filtrer intelligemment les documents injectés dans les prompts IA.

📌
Règle de filtrage
Un document avec des tableaux vides (matieres: [], niveaux: []) est considéré comme général et est toujours inclus. Un document avec des valeurs non vides n'est inclus que si elles correspondent au profil de l'élève. Implémenté avec OR: [{ matieres: { isEmpty: true } }, { matieres: { has: matiere } }] dans Prisma.

Interface d'administration : src/components/admin/gerer-documents-client.tsx. Les boutons toggle permettent d'activer/désactiver chaque matière et chaque niveau (groupés en Primaire 1–6 et Secondaire 1–5). Fichier de filtrage : src/lib/ai/contexte-documents.ts.

Choix du niveau de difficulté par l'élève

Avant de générer un exercice ou une épreuve, l'élève peut choisir parmi trois niveaux de difficulté. Ce choix remplace le calcul automatique basé sur le niveau actuel et l'état émotionnel.

Valeur (NiveauDifficulte)LabelComportement IA
BASE😊 FacileExercice accessible, vocabulaire simplifié, moins d'étapes
ATTENDU🎯 IntermédiaireNiveau standard du curriculum (valeur par défaut)
AVANCE🚀 DifficileExercice enrichi, problèmes composés, raisonnement plus profond

L'étape "Difficulté" s'insère dans le wizard élève entre "Notions" et "Durée" (pour les épreuves) ou directement avant la génération (pour les exercices simples). Fichier UI : src/app/(dashboard)/eleve/exercices/nouveau/page.tsx. Côté IA : difficulteChoisie est passé à generateExercice() et genererEpreuve() dans src/lib/ai/exercice.ts, où il remplace getDifficulteAdaptee() si fourni.

Écrans de chargement animés (génération et correction)

Les attentes IA (génération d'exercice, génération d'épreuve, correction d'épreuve) affichent un écran de chargement animé adapté pour les enfants, au lieu d'un simple spinner.

Architecture commune

  • Barre de progression circulaire SVG — cercle avec stroke-dashoffset animé via CSS transition. Progresse de 0 à 100 % sur la durée estimée.
  • Messages rotatifs — tableau de messages enfant-friendly, rotation toutes les 3,5 s via setInterval. Chaque changement déclenche un fadeIn CSS via key={msgIndex}.
  • Notification sonore — accord ascendant C5→E5→G5→C6 généré par Web Audio API (AudioContext + OscillatorNode + GainNode). Silencieux si AudioContext indisponible (SSR, restrictions navigateur).

Système de surprises parentales

Les parents peuvent accorder des récompenses réelles (privilèges) à leurs enfants, déclenchées automatiquement par des jalons comportementaux. Maximum 7 par an, complètement invisible pour l'enfant jusqu'au moment de la révélation.

Cycle de vie

prisma
// StatutSurprise enum
EN_ATTENTE  → jalon détecté, parent notifié, en attente de son choix
ACCORDE     → parent a choisi un privilège, enfant notifié
CONFIRME    → enfant a découvert, écrit un souvenir, surprise archivée
EXPIRE      → parent n'a pas agi dans les 14 jours (auto via updateMany)

Détection des jalons

Appelé dans un bloc waitUntil() après chaque soumission d'exercice. Fichier : src/lib/surprises/detecter-jalon.ts.

JalonCondition
Streak multiple de 7streakJours % 7 === 0 && streakJours > 0
Score parfait après 2+ tentativesscore === 100 && nbTentatives >= 2
Progression +20 ptsScore actuel ≥ dernier score + 20
Streak 21 joursstreakJours === 21

Quotas vérifiés avant création : max 7 surprises dans l'année civile + délai minimum 30 jours depuis la dernière surprise confirmée.

Génération des options

Fichier : src/lib/surprises/generer-options.ts. Basé sur des templates (pas d'appel IA) : dictionnaire d'options indexé par centre d'intérêt (SOCCER, HOCKEY, JEUX_VIDEO, MANGA, etc.). Retourne exactement 3 options personnalisées. Fallback sur 8 options génériques si les intérêts de l'enfant ne matchent pas le dictionnaire.

Email parent

Fichier : src/lib/email/send-surprise-parent.ts. Envoyé via Resend quand une surprise passe en EN_ATTENTE. Sujet : "🌟 [prénom] mérite une surprise !". En mode DEV (RESEND_API_KEY absente) : console.log sans erreur.

Endpoints tRPC

ProcédureRôleQui peut l'appeler
surprise.listerParentLit les surprises EN_ATTENTE/ACCORDE + expire les anciennesParent

⚠️
Invisible jusqu'à l'accord
L'enfant ne voit JAMAIS qu'un jalon a été atteint. La carte surprise n'apparaît sur son dashboard que lorsque le parent a explicitement accordé la surprise (statut ACCORDE). Si le parent ignore la notification, l'enfant ne sait jamais qu'une surprise était possible.

Points de vigilance importants

⚠️
Tâche en suspens — Suivi du temps (sessions_pratique) La table sessions_pratique existe dans le schéma Prisma mais aucun code ne crée de sessions. Le tableau de bord parent affiche toujours 0 minute. Il faudra implémenter un mécanisme de heartbeat dans le dashboard élève pour peupler cette table.
⚠️
Tâche en suspens — Date de naissance des élèves existants Les élèves déjà inscrits n'ont pas de date de naissance renseignée. Une interface dans /eleve/parametres devrait permettre de la saisir (nécessaire pour les fonctionnalités liées aux anniversaires).
🔌
Connexion à la DB en local vs Vercel En local, utiliser le port 5432 (connexion directe Supabase). Sur Vercel, utiliser le port 6543 (PgBouncer, mode Transaction). La variable DATABASE_URL dans Vercel doit pointer vers le port 6543, et DIRECT_URL vers 5432.
